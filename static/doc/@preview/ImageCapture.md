The **Image Capture** operation converts visual HTML output from preceding steps into a cropped PNG image asset.

The asset is encoded back into the data model as an ASP atom using the predicate specified by `output_predicate` (defaults to `__png__/1`). The single term contains the base64-encoded string of the captured image.

Properties can be configured directly through the UI inputs to define crop boundaries and auto-execution delays.

§§§§

This operation captures rendered HTML components dynamically and updates your ASP program with the resulting image data.

#### Image Storage

`OUTPUT_PREDICATE("BASE64_STRING")`

Example output atom generated by the recipe:
```asp
__png__("iVBORw0KGgoAAAANSUhEUgAA...").
```

#### Cropping Content

Define integer pixel values in the component UI to crop edges and focus on specific data:
*   **Left / Right**: Removes pixels from the horizontal boundaries.
*   **Top / Bottom**: Removes pixels from the vertical boundaries.

### Interaction and Capture Lifecycle

*   **Target Selector**: Automatically finds the closest preceding element with a side output.
*   **Manual Trigger**: Click **Store Image** to take an instant snapshot of the current view.
*   **Auto Capture**: Enable **Auto** and set a delay (e.g., `100` ms) to let the recipe capture the output layout automatically upon execution.
*   **Clear State**: Click **Clear Image** to wipe the current snapshot and reset the predicate binding.

#### Tip: Using Captured Images in Later Steps

You can pass the generated base64 string to subsequent ingredients (like custom markdown or report blocks) to render saved program states:
```asp
{{= png(X) : __png__(Data) }}
```
