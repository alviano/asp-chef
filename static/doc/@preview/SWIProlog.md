The **SWI Prolog** operation executes an embedded SWI-Prolog program and evaluates queries against it directly inside the browser using WebAssembly.

Program and queries are provided by Base64-encoded predicates, possibly using Mustache queries.
Facts from the input model are included in the Prolog program, allowing you to reason over the results of your ASP models with the full power of Prolog's logic programming capabilities.

The output is generated as new facts in the model, using the `__answer__/*` predicate. Each term in the predicate corresponds to a variable substitution, with the variables ordered alphabetically by their names.
- Strings are double-quoted.
- Non-integral numbers are wrapped in `real("NUMBER")` (e.g., `real("3.14")`).

The input is echoed in output.

§§§§

This operation allows you to run deterministic Prolog reasoning, list manipulation, or mathematical calculations directly over the results or structures generated by your ASP models.

#### How It Works
1.  **Extraction**: The chef looks for the `__program__("BASE64")` and `__query__("BASE64")` atoms in your input model.
2.  **Decoding**: It decodes the Base64 payloads into raw Prolog code text and query strings. Mustache queries are evaluated at this stage, allowing you to inject dynamic content from your ASP model into the Prolog code.
3.  **Execution**: The underlying WASM engine loads the program and runs the query.
4.  **Assertion**: Every found solution is mapped back into the output model using the `__answer__` predicate structure.

#### Basic Integration Example
Suppose your input ASP model contains a Base64-encoded logic theory defining family relations and a target query.
**Example input model:**
```asp
% Program:
%   parent(orazio, claudio).  % alternative: put this fact in the input model
%   parent(claudio, marco).   % alternative: put this fact in the input model
%   grand_parent(X, Y) :- parent(X, Z), parent(Z, Y).
__program__("cGFyZW50KG9yYXppbywgY2xhdWRpbykuCnBhcmVudChjbGF1ZGlvLCBtYXJjbykuCmdyYW5kX3BhcmVudChYLCBZKSA6LSBwYXJlbnQoWCwgWiksIHBhcmVudChaLCBZKS4=").

% Query:
%   grand_parent(GrandParent, Parent).
__query__("Z3JhbmRfcGFyZW50KEdyYW5kUGFyZW50LCBQYXJlbnQpLg==").
```
**Resulting Output Model Atoms:**
```asp
__answer__("orazio","marco").
```

If the query produces multiple valid substitutions (e.g., if Horace had multiple grandchildren), a separate `__answer__` atom is generated for every discovered solution.

Also note that queries can be more complex, involving multiple variables, atoms and conditions.
Terms in each `__answer__` atom are ordered alphabetically by variable name, so the first term corresponds to the variable that comes first in alphabetical order.
For example, if the query was `grand_parent(X, Parent).`, the output would be:
```asp
__answer__("marco","orazio").
```

#### Using Mustache Templates

You can make your Prolog logic highly dynamic by embedding Mustache expressions inside your Base64-encoded strings. When multistage is enabled, data from other predicates in your ASP model can be injected straight into your Prolog facts or queries before the SWI engine executes them.
