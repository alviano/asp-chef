The **Unique Consecutive** operation removes each model that is **immediately** preceded by another copy of the model.

Each model is compared with the previous model and provided in output only if they are different. A list of predicates can be ignored.

Note that order of elements in models does matter, that is, models with the same elements but in different orders will be considered different models. A **Sort Canonical** operation can be used to make the ordering immaterial. A **Sort Models Canonically** operation can be used to obtain a behavior analogous to the **Unique** operation (even if it would make more sense to opt for the **Unique** operation in this case).

§§§§

This operation is useful for de-duplicating consecutive identical models, often generated by repeated searches or iterative operations.

#### Why Use Unique Consecutive?

If you have a large list of models (e.g., from an **Optimize** operation that returns many results during its run) and you only care about changes between consecutive models:
- Model 1: `{ a., b. }`
- Model 2: `{ a., b. }` (Discarded)
- Model 3: `{ a., c. }` (Kept)

#### Ignoring Predicates

By specifying a list of **Ignore Predicates** (e.g., `__base64__`, `__costs__`), you can compare models based on their meaningful domain facts while ignoring internal markers or metadata.

#### Differences with Unique Operation

- **Unique Consecutive**: Only checks the *immediately* preceding model. Much more efficient for large streams.
- **Unique**: Checks *all* previous models encountered in the entire sequence.

Use **Unique Consecutive** when your process generates "bursts" of duplicate models or when you only want to see a value change (like a state machine transition).
